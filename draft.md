# AI Callcenter

## 概要
Azure HorizonDBを中核に据えた、AIにより自動化されたコールセンターのPoCを実装する。

HorizonDBのドキュメント一式は以下にあるので、実装の際はこの一次ソースで必ず確認しながら作業を進めること。
https://learn.microsoft.com/ja-jp/azure/horizondb/

## 要件

- 顧客は電話でコールセンターに問い合わせを行う。
- コールセンターは、電話音声をリアルタイムでテキストに変換する。Azure上で音声データをリアルタイムにパイプラインに流し込み、同じくリアルタイムでText to Speechを行えるFoundry上のLLMを用いて、テキスト化する。
- リアルタイムにテキスト化するので、テキストはストリーム形式になる。このストリームの処理と応答を返すのに適切なAzure上のアプリケーションプラットフォームを選択する。Azure FuctionsやContainer Appsなどが候補となる。
- ストリーミングされるテキストを、HorizonDBにUPSERTで更新し続ける。
- HorizonDBには、会話IDを主キーとするテーブルがあり、シーケンス番号とテキストと埋め込みベクトルを格納できる。一定間隔、あるいは顧客の発言が途切れたタイミングで、テキストはtext-embedding-3-largeを用いて埋め込みベクトル化され、同じテーブルに格納される。このタイミングでシーケンス番号はインクリメントされる。
- HorizonDBには想定応答のマスターデータがあり、ID、応答のテキスト、埋め込みベクトルが記録されている。
- 上記、顧客の発言が途切れたタイミングで生成された顧客の発言の埋め込みベクトルと、想定応答のマスターデータをコサイン距離で比較して、最も近い応答を選択する。選択された応答が複数ある場合、rerankerによって並び替え、応答精度を向上させる。
- 選択された応答の最上位のテキストを、Foundry上のLLMに渡して、Text to Speechを行い、顧客に返答する。
- HorizonDBを含む全てのリソースは、Azure Developer CLI (azd) を用いて管理する。リソースのデプロイはazd upコマンドで実行でき、デプロイするリージョンの選択とリソースグループの新規作成・選択以外、secretの生成やHorizonDBのエクステンションの有効化など、必要な作業は全てazd upコマンドで自動的に行われる。
- この実装では、Azure MCP Serverのtoolsによって提供される知見を活用する。
- 実装作業の進捗は、Markdown形式のドキュメントに記録すること。
- 開発・デバッグ中の顧客の音声入力は、開発環境で利用できるUSBマイクなどで代替する。このために必要なアダプターをSwift/C#などで実装すること。
